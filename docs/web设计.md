二级页面：
数字资源管理：

- 数字资源列表：显示一个资源列表。上面有一个切换链的按钮，可以浏览其他链的资源。可以点进去，跳转数字资源管理，显示资源详细信息，点进去后可能可以修改内容。
- 数字资源创建：
  资源名称/资源元数据/资源类别（commitment/文件/列表）
  资源内容
  资源权限：读权限（abe读加密/跨链约束）/修改权限/添加权限/删除权限
  [创建资源] 创建相关的合约
- 权限预设样板：一个权限预设样板列表，存储在web端本地。每个权限就是一段简短的代码。支持写代码创建新的权限（可组合/复杂访问控制）
- 数字资源管理：展示资源内容，如果是abe读加密的，则需先显示一个解密按钮，点击后进行解密。根据权限判断能否修改（要不要做成都可以点，但是点了后会警告，如果不满足则交易失败？ 以凸显跨链）

跨链隐私计算： 管理合约只部署到一个链上，但通过一个interface合约支持跨链调用。

- 计算任务列表：展示所有任务列表及任务信息
- 创建计算任务：创建新任务。需要指定任务使用的资源（链id+地址），任务参与方需要满足的条件（对应跨链身份验证），计算任务模板。
- 执行计算任务：输入任务id。如果是对应参与方，则先需要发起确认任务交易，该交易包含身份证明、输入承诺和加密公钥。所有参与方确认后，出现[开始执行]按钮。执行完毕后，发起方出现[链上提交]按钮，更新链上任务状态和结果摘要。
- 计算模板管理：一堆列表/card和一个大输入框。展示mpc的那个代码。有预设模板，可以添加新的。

隐私身份管理：\*监管方1：

- 凭证颁发：（该部分上链commit/可以不上链？ 主要做不可连接性）
  网页生成链下凭证（姓名/性别/住址/出生日期/单位/学历/职务/有效期 etc）+监管方签名（签名私钥放后端）
- 身份恢复与追溯：
  事件列表（上方有搜索框）-事件列表可点入
  点入后，显示事件详细信息-按钮[身份恢复]（点击后出现一个钱包签名，之后把签名传到后端做简单验证'假的'，后端计数+1）
  在计数足够后，后端执行签名恢复流程。之后事件详细信息页显示原始身份。按钮[历史追溯]，点入后显示一个完整的该用户的事件列表。

\*监管方2:

- 身份恢复与追溯：
  同上

\*用户:

- 凭证信息：把监管方生成的导入浏览器存储，之后该页面变成凭证页面。
- 凭证使用：负责生成相关证明
  需要选择：选择性暴露身份内容/或预制证明模板（例如证明年龄>=18 不是医生/是医生/在xx工作）
  无效标识符（抗女巫攻击）。
  按钮[生成证明]，生成一个proof文本框
- 凭证验证：
  一个文本框，输入proof点击后可以验证是否是合理的凭证，以及proof中暴露的信息
  当前方案缺陷：未体现跨链，是否需要强行缝？

医疗管理应用：

- 医生身份登记：
  按钮[生成凭证]：跳转生成 选择性暴露姓名+单位+学历+职务proof 选择科室 按钮[注册身份] 此处可以示例抗女巫攻击。
- 医院病例管理：
  选择相关病例资源后，跳转到数字资源管理。
- 多院病例共享：
  两个预设任务：相同患者求交/病例症状统计。点击后，选择需要进行计算的计算资源，之后跳转到跨链隐私计算页面。
