## 一、整体设计

本项目面向跨链环境下的数据安全访问、隐私身份认证和隐私计算需求，建设一套“跨链数据安全及隐私身份平台”。系统以以太坊兼容区块链为基础，结合跨链消息桥、去中心化身份、链上访问控制、属性加密和链下隐私计算等技术，实现多链环境下的数据资源可信访问、身份隐私保护和跨链协同计算。

系统整体由三类应用平台组成：

1. **支持跨链的区块链隐私计算平台**
   实现跨链计算任务发起、参与方确认、链下隐私计算、链上结果记录与审计。
2. **跨链身份隐私方案和应用**
   实现用户隐私身份注册、凭证签发、选择性披露、不可链接认证、抗女巫攻击、身份恢复和合规追溯。
3. **访问控制应用框架及实例应用**
   实现链上数字内容资源注册、访问策略配置、跨链访问请求、属性加密保护和授权审计。

三类系统通过统一的跨链消息接口进行协同。身份系统负责提供可信身份属性，访问控制系统负责判断是否具备资源访问权限，隐私计算平台负责在不暴露原始数据的情况下完成跨链协同计算。

## 二、系统支持功能

### 1. 跨链基础支撑能力

系统应支持多条以太坊兼容链之间的跨链通信，包括：

- 支持多链部署与链间消息传递（跨链桥底层合约）；
- 支持链 A 调用链 B 上的业务合约（业务合约统一接口，下同）；
- 支持跨链请求、跨链回执、跨链结果返回；
- 支持跨链消息的来源验证、防重放和状态记录；
- 支持业务合约识别真实调用方，而不是直接依赖 `msg.sender`；
- 支持隐私计算任务、身份验证结果、访问控制请求等业务消息跨链传递；

跨链部分作为三个业务系统的公共底座。

# 三、模块设计

## 模块一：跨链基础平台

### 1. 模块目标

建设统一的跨链通信底座，为隐私计算、隐私身份和访问控制应用提供跨链消息传递能力。

### 2. 技术方案

跨链平台采用**基于 Oracle/Relayer 的跨链消息桥方案**。每条链上部署跨链网关合约，链下 relayer 监听源链事件，将消息提交到目标链。目标链验证消息来源、消息序号、签名和目标合约后，调用对应业务合约。

技术路线如下：

```
源链业务合约
  → CrossChainGateway 发送跨链消息
  → Relayer/Oracle 监听链上事件
  → 目标链 CrossChainGateway 验证消息
  → 调用目标链业务合约
  → 跨链返回执行结果
```

### 3. 关键技术点

- 设计统一跨链消息格式；
- 支持跨链消息 nonce，防止重放攻击；
- 支持可信源链和可信源合约白名单；
- 支持跨链请求状态管理；
- 支持跨链回调；
- 支持业务合约统一获取真实调用者。

通过设计一套跨链上下文库，使得支持跨链的合约可以便捷接入。跨链调用上下文建议设计为：

```
本链调用：sender = msg.sender
跨链调用：sender = 源链真实调用者
```

业务合约中不直接使用 `msg.sender`，而使用统一的跨链上下文接口获取调用者。

### 4. 主要子模块

| 子模块                                 | 说明                                                                                                                   |
| -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| 跨链网关底层合约                       | 负责跨链消息发送、接收和验证，提供跨链桥的消息中继功能                                                                 |
| Relayer/Oracle 服务                    | 负责监听源链事件并向目标链提交消息                                                                                     |
| 跨链上下文库                           | 为业务合约提供统一调用方识别接口                                                                                       |
| 消息状态管理模块合约                   | 记录消息发送、接收、执行和回调状态，在跨链桥底层抽象的消息中继功能上做开发，保证就接口的统一性（即跨链桥架构可迁移性） |
| 可信源管理模块（放到状态管理模块里面） | 管理可信链、可信合约和可信 relayer                                                                                     |
|                                        |                                                                                                                        |

### 5. 交付物

- 多链测试环境；
- 跨链网关合约；
- Relayer/Oracle 服务；
- 跨链上下文合约库；
- 跨链调用文档；
- 跨链消息测试用例；
- 跨链示例demo。

## 模块二：跨链隐私计算平台

### 1. 模块目标

建设一套支持跨链任务发起、链下 MPC 计算和结果跨链返回的隐私计算平台。

### 2. 技术方案

隐私计算采用**链上任务编排 + 链下 MPC 计算**的模式。区块链负责计算任务登记、参与方确认、任务状态维护和结果摘要记录；真实计算过程在链下/链上 MPC 网络中完成。

采用成熟 MPC 框架MP-SPDZ为基础。

技术路线如下：

```
用户发起隐私计算任务
  → 链上记录任务信息
- 任务信息需要对应"跨链资源"这一概念，通过引入资源注册合约、资源metadata、资源内容承诺等方式实现。该部分合约具体实现由模块3支持。
  → 跨链通知其他参与方
  → 各方确认参与并进行链上密钥和会话信息交换
  → 链下 MPC 节点建立计算会话
  → 执行加法、乘法或业务计算模板
  → 结果跨链返回请求方
```

### 3. 关键技术点

- 使用秘密分享支持多方输入保护；
- 使用SPDZ 类协议支持乘法计算等通用计算；
- 支持加法、乘法和固定业务计算模板；
- 支持针对PSI问题的高效MPC计算算法；
- 链上验证参与方签名和任务状态；
- 支持跨链发起任务和跨链返回计算结果；
- 对常用计算函数进行模板化，保证性能和可控性。

### 4. 支持计算类型

参考类型：

```
1. 多方加法求和；
2. 多方乘法；
3. 加权评分计算；
4. 统计聚合；
5. 简单条件判断；
6. 隐私集合求交
```

### 5. 主要子模块

| 子模块           | 说明                                      |
| ---------------- | ----------------------------------------- |
| 计算任务管理合约 | 创建任务、记录状态、管理参与方            |
| 计算函数注册模块 | 注册支持的计算函数和参数格式              |
| 参与方管理模块   | 管理参与计算的机构、节点和权限（结合DID） |
| MPC 链下计算服务 | 调用 MPC 框架完成实际计算                 |
| 输入承诺模块     | 记录输入哈希或承诺，避免输入泄露          |
| 结果提交模块     | 提交结果、签名和计算摘要                  |
| 跨链计算适配模块 | 支持跨链任务请求和结果返回                |
| 审计查询模块     | 查询计算状态、参与方和结果记录            |

### 6. 交付物

- 隐私计算任务合约；
- MPC 计算服务；
- 加法、乘法计算模板；
- 至少三个业务计算模板；
- 跨链计算适配接口；
- 计算任务 API；
- 隐私计算前端展示页面/demo；

## 模块三：跨链隐私身份系统

### 1. 模块目标

建设一套可跨链使用的隐私身份系统，实现用户身份属性证明、选择性披露、不可链接认证、抗女巫攻击、身份恢复和合规追溯。

### 2. 技术方案

基于实验室之前的项目开发，现在需要确认上述目标是否都在该系统中有支持。
https://github.com/didnet/zxDID

另外 考虑确认是否支持去中心化的凭证颁发方案（如果没有的话，能否使用mpc？）

身份系统采用**DID + VC + ZKP + 追溯授权**的技术路线，代码基于开放链项目开发。

- DID 用于表示用户、机构、数据资源和监管方身份；
- VC 用于表达用户身份属性，例如医生资格、科研资质、政务审批资格；
- ZKP 用于实现选择性披露，只证明满足条件，不暴露完整身份；
- 身份恢复通过 guardian 或多方阈值机制实现；
- 身份追溯通过监管授权和审计流程实现。

技术路线如下：

```
机构注册 DID
  → 机构为用户签发身份凭证
  → 用户本地保存凭证
  → 用户访问资源或发起计算时生成 ZK 证明
  → 链上验证证明和凭证状态
  → 支持跨链传递身份验证结果
```

### 3. 关键技术点

- DID 注册与 DID Document 管理；
- VC 凭证签发、验证、撤销；
- 基于 ZKP 的属性证明；
- 凭证选择性披露；
- 应用相关 nullifier，防止跨场景关联；
- 跨链同步凭证状态或验证结果；
- 多 guardian 身份恢复；
- 监管阈值授权追溯真实身份；
- 追溯过程上链审计。

### 4. 典型证明能力

系统应支持用户证明：

```
我是认证医生，但不暴露姓名；
我属于某认证医院，但不暴露工号；
我具有科研审批资格，但不暴露完整凭证；
我没有重复申请，但不同业务之间不可链接；
我的凭证有效且未撤销。
```

### 5. 主要子模块

| 子模块             | 说明                           |
| ------------------ | ------------------------------ |
| DID 注册合约       | 管理用户、机构、监管方 DID     |
| 机构管理模块       | 管理可签发凭证的可信机构       |
| 凭证签发模块       | 为用户签发身份属性凭证         |
| 凭证状态模块       | 管理凭证有效、过期、撤销状态   |

## 模块四：访问控制应用框架

### 1. 模块目标

建设一套面向链上数字内容和跨链资源的访问控制框架，支持复杂策略组合、跨链访问授权和 CP-ABE 数据保护。

### 2. 技术方案

访问控制采用**链上策略控制 + CP-ABE 属性加密 + 链下密文存储**的技术路线。

- 链上策略控制负责判断用户是否具备访问资格；
- CP-ABE 负责保护数据密钥；
- 原始数据存储在线下数据库、对象存储中；
- 链上记录资源哈希、策略方案、存储地址和授权日志；
- 用户通过 DID/VC/ZKP 证明身份属性；
- 策略引擎判断是否授权；
- 只有满足 CP-ABE 属性策略的用户才能解密数据密钥。

技术路线如下：

```
资源方注册数据资源
  → 上传密文数据到链下存储
  → 数据密钥使用 CP-ABE 加密
  → 链上登记资源哈希、策略和存储地址
  → 用户提交访问申请和身份属性证明
  → 链上策略引擎判断权限
  → 授权用户获取密文和加密密钥
  → 用户本地解密数据
  → 访问行为上链审计
```

### 3. 关键技术点

- 链上访问策略配置；
- 与隐私身份系统对接，验证用户属性证明；
- CP-ABE 对数据密钥进行加密；
- 数据密文链下存储；
- 链上只存哈希、策略和审计记录；
- 支持跨链访问申请；
- 支持跨链授权结果返回；
- 支持访问日志审计和违规追溯。

### 4. 策略类型

访问控制框架应支持：

```
1. 基于角色的策略：医生、科研人员、监管人员；
2. 基于属性的策略：科室、机构、资质、等级；
3. 基于用途的策略：诊疗、科研、审批、监管；
4. 基于时间的策略：有效期、访问窗口；
5. 基于审批的策略：伦理审批、监管审批；
6. 基于跨链来源的策略：可信链、可信合约、可信机构；
7. 组合策略：AND、OR、阈值门限、多条件嵌套。
```

示例策略：

```
医生 AND 肿瘤科 AND 认证医院

或者

科研人员 AND 伦理审批通过 AND 用途为科研 AND 数据范围为脱敏数据
```

### 5. 主要子模块

| 子模块                | 说明                                               |
| --------------------- | -------------------------------------------------- |
| 资源注册模块          | 登记资源、哈希、所有者、存储地址                   |
| 策略管理模块          | 创建、修改和查询使用访问策略（通过创建子合约实现） |
| CP-ABE 加密模块(链下) | 对数据密钥进行属性加密                             |
| 数据获取模块          | 对接链下密文存储                                   |
| 访问审计模块          | 记录访问申请、授权、拒绝和下载行为                 |

### 6. 交付物

- 资源注册合约；
- 策略管理合约；
- CP-ABE 加解密服务；
- 数据资源管理 API；

- 访问控制前端页面/demo；

## Demo 1：跨链医疗数据安全共享与隐私计算

### 场景设计

构建医院链、科研机构链和监管链。医院链保存医疗数据资源，科研机构链发起访问和计算请求，监管链负责身份凭证和追溯审计。

### 展示功能

```
1. 医生或科研人员通过隐私身份完成认证；
2. 科研机构跨链申请访问医院数据；
3. 医院链根据访问策略进行授权；
4. 数据密钥采用 CP-ABE 加密；
5. 多家医院通过 MPC 完成病例数量统计或风险评分；
6. 计算结果返回科研机构链；
7. 全流程记录访问和计算审计日志；
8. 异常访问可触发身份追溯。
```

### 对应展示技术

```
跨链消息；
隐私身份认证；
选择性披露；
链上访问控制；
CP-ABE 数据保护；
MPC 加法和乘法；
跨链结果返回；
身份追溯。
```
